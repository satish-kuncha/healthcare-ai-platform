# Project Vision

## Mission

Learn AI Application Engineering by evolving a simple
PydanticAI healthcare assistant into a production-grade
AI platform.

This project is primarily a learning vehicle and secondarily
an architecture demonstrator.

---

## Primary Learning Goals

### Agent Engineering

- PydanticAI
- Structured Outputs
- Tool Calling
- Dependency Injection
- Async Agents

### Workflow Engineering

- LangGraph
- State Machines
- Branching
- Checkpointing
- Human-in-the-Loop

### AI Application Engineering

- Rules
- Guardrails
- Memory
- State Management
- Evaluations
- Observability

### Knowledge Systems

- Enterprise RAG
- Hybrid Search
- Metadata Filtering
- Reranking
- Retrieval Evaluation

### Integration

- MCP
- Service Boundaries
- Tool Abstraction

### Backend

- FastAPI
- Async Python
- PostgreSQL

### Frontend

- React
- Voice Interface

### Cloud

- Docker
- ECS Fargate
- ALB
- RDS
- CloudWatch
- Secrets Manager

---

## Guiding Philosophy

Workflow controls AI.

AI does not control workflow.

Business rules are deterministic.

AI performs reasoning and interpretation.

---

## End State

Healthcare Operations Platform demonstrating
modern AI application architecture end-to-end.
